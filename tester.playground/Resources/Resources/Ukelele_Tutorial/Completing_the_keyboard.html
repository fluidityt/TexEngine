Completing the keyboard layout

There are two other dead keys to create. The procedure is exactly the same for them as it was for the first dead key, beginning with selecting “Create Dead Key…” from the Keyboard menu, or typing ⌘-D, or clicking the Create button on the toolbar. Again, we want to create a new dead key, so we type the dead key, and choose the first option in the dialog. As an alternative to typing the dead key, you can also click the key in the main window, with the appropriate modifiers down.

Again, for each of the new dead keys, add the output for the appropriate keys, and then select “Leave Dead Key State” from the Keyboard menu, or click the Leave button in the toolbar.

The keyboard layout is now complete, so all that is required is to save it and install it. Both can be done in one step by saving the keyboard layout in the appropriate Keyboard Layouts folder (usually in either the Library folder of your home directory or in the Library folder at the top level of your startup disk). It is best to give the name of the keyboard layout for the file name, so we will call it Yolngu. The extension “.keylayout” should be added automatically, and is usually hidden.


Now you need to test the keyboard layout. First, log out and log in again. Next, open the International pane of System Preferences, and select the Input tab. Find Yolngu in the list, and click the checkbox so that it is checked.


Now, Yolngu should appear in the Input menu. You should be able to select it and use it from this point.


That is it! There are more things that can be done with Ukelele, and some other ways of doing some of the tasks covered in this tutorial. For more details, continue with Part Two of this tutorial, or see the online help and the manual.
